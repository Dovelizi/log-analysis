## Dashboard 慢点定位
- ✅ 慢点接口: /api/dashboard/hitch-control-cost-time/statistics 5399ms
- ✅ 根因不是缺索引 (5 张表 create_time 索引齐全)，是数据倾斜 + Buffer Pool 不足:
  - hitch_control_cost_time 单日 (2026-04-26) 29.3 万条 = 全表 90%+
  - 接口串行调 9 条 SQL，每条都扫 25 万行索引
  - InnoDB Buffer Pool=128MB 装不下 224MB 表 → 磁盘 IO 拖慢
- 优化方向: SQL 合并 (9→1) 必做; Buffer Pool 扩容/Redis 缓存 待用户拍板
